If you too have felt a little confused by the many different interfaces that Codex offers and wondered what to use when, this decision tree should make it pretty straight forward. 

![](/Codex-Decision-Tree.png)

Keep in mind though that technically you can still work on multiple things simultaneously with IDE extensions too, just have multiple sessions running. Each task does get its own agent and sandbox.

PS 1 - For what its worth, this same graphic should also be translatable to Claude Code. As of now, Claude Code doesn't offer a macOS app so omit that part of the flowchart, but a Claude Code Web interface is indeed in [beta](https://code.claude.com/docs/en/claude-code-on-the-web) so you can substitute Codex Web with Claude Code Web.

PS 2 - These capabilities keep evolving. For example, there is now even a dropdown in the IDE extension that lets you specify that the task should be run in cloud.
